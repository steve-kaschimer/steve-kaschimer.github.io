# Kiro Hooks

This folder is intentionally scaffolded with guidance only.

Kiro supports trigger-based automations ("hooks") that can run around common workflows.

## What To Use This For

Use hooks for repeatable automations, such as:

- validating pre-implementation context
- running checks before completion
- enforcing project standards
- capturing audit/review signals

## How To Start

1. Review current Kiro docs/changelog for hook capabilities.
2. Add one low-risk hook and confirm behavior in your normal workflow.
3. Keep hook logic aligned with `AGENTS.md` and project constitution.
